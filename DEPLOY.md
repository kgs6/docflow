# Запуск проекта DocFlow

Этот проект использует Next.js, Prisma и SQLite (через libSQL).

## Предварительные требования

- Node.js (рекомендуется v20+)
- npm или другой менеджер пакетов (pnpm, yarn)

## Установка

1. Установите зависимости:
   ```bash
   npm install
   ```

2. Настройте переменные окружения:
   Создайте файл `.env` (если его нет) на основе примера. Для SQLite достаточно указать путь к базе:
   ```env
   DATABASE_URL="file:./dev.db"
   ```

3. Инициализируйте базу данных (создание таблиц):
   ```bash
   npx prisma migrate dev
   ```

4. (Опционально) Заполните базу начальными данными:
   ```bash
   npx prisma db seed
   ```

## Запуск на сервере

### Режим разработки
```bash
npm run dev
```

### Сборка и запуск в продакшн-режиме
1. Соберите проект:
   ```bash
   npm run build
   ```
2. Запустите сервер:
   ```bash
   npm start
   ```

### Особенности работы с SQLite на сервере
- Убедитесь, что файл `dev.db` доступен для чтения и записи процессу Node.js.
- Для продакшн-серверов рекомендуется использовать файловую систему с хорошей поддержкой конкурентной записи (например, ext4).
- Если вы используете облачные сервисы (VPS), убедитесь, что путь к `dev.db` указан корректно в `DATABASE_URL`.
